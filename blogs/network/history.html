    # 浏览器历史记录

    # 如何生成一条历史记录？

    • 直接在当前页面的地址栏中输入地址
    • 点击页面中有a标签的href
    • 执行location.href = ‘xxx’(location.replace(‘xxx’)生成一条记录取代当前指针所指向的记录)
    • 表单提交跳转（注意只能跳到当前窗口）
    • 使用pushState方法可以不刷新页面就可以生成一条历史记录，页面URL发生改变
    • repalceState 替换当前历史记录，不产生新的历史记录

    # 管理历史记录

    历史记录在浏览器中以栈的形式进行管理，每次增加的历史记录都会在栈的最顶端，以谷歌浏览器为例。

    <button id='btn'> pushState /path</button>
<button id='btn2'> pushState #/a</button>
<button id='btn4'> pushState ?id=190</button>
<button id='btn5'> replaceState #/b</button>
...

var btn = document.getElementById('btn')
btn.onclick =() => {
    window.history.pushState({a: 1}, 'title', '/path');
    console.log(history)
}
var btn2 = document.getElementById('btn2')
btn2.onclick =() => {
    history.pushState({a: 1}, '', '#/a');
}
var btn4 = document.getElementById('btn4')
btn4.onclick =() => {
    history.pushState({a: 4}, '', '?id=190');
} 
var btn5 = document.getElementById('btn5')
btn5.onclick =() => {
    history.replaceState({a: 1}, '', '#/b');
}

    # History API

    下面三个不会生产新的历史记录，只是栈指针的移动。

    • forward()
    // 历史记录指针向下移动一位。如果已经在栈低，指针会移动到栈定
history.forward(-1) // -2 -3
// 地址栏变为 http://localhost:3001/path#/a 栈指针指向 /path#/a

history.go()

    # onhashchange

    当 一个窗口的 hash （URL 中 # 后面的部分）改变时就会触发 hashchange 事件。

    # 什么是hash？

    hash即URL中"#"字符后面的部分。

    1. 使用浏览器访问网页时，如果网页URL中带有hash，页面就会定位到id（或name）与hash值一样的元素的位置；

    2. hash还有另一个特点，它的改变不会导致页面重新加载；

    3. hash值浏览器是不会随请求发送到服务器端的；

    4. 通过window.location.hash属性获取和设置hash值。

    window.location.hash值的变化会直接反应到浏览器地址栏（#后面的部分会发生变化），同时，浏览器地址栏hash值的变化也会触发window.location.hash值的变化，从而触发onhashchange事件。

    # onhashchange触发的时机。

    由以下行为引起的hash的改变会触发onhashchange事件。

    1. 浏览器的前进，后退
    2. history的go back forward 操作

    pushState和replaceState操作都不会触发onhashchange事件。

    实际上历史记录必须是history.pushState()之后形成的。前进到这条记录，或者后退到这条以前的记录，如果hash发送了变化，都会触发onhashchange事件，同时也会触发浏onpopstate事件。

    # onpopstate

    地址栏发生改变，满足一定的条件下会触发onpopstate事件。

    history.pushState()或history.replaceState()也可以实现地址栏改变浏览器不刷新的效果。 但是history.pushState()或history.replaceState()不会触发popstate事件。

    # onpopstate的触发时机

    执行了history.pushState()或history.replaceState()之后再进行下面的操作。

    前进或者后退到的状态必须是history.pushState()或history.replaceState()之后形成的。

    1. 点击浏览器的前进按钮和后退按钮
    2. 执行js代码history.go(n)/history.forward()/history.back()

    如果这么说不好理解，下面就具体给大家分析下：

    场景1

    • 当前地址 http://localhost:3001

    • 使用window.history.pushState("", "", '?id=190')后，地址栏变为http://localhost:3001?id=190（这个过程不会触发onpopstate事件）

    • 点击浏览的返回按钮或者js执行history.back()等操作，地址栏从http://localhost:3001?id=190。变为http://localhost:3001，触发onpopstate事件。

    场景2

    • 当前地址 http://localhost:3001。

    • 利用a标签跳转到http://localhost:3001?id=190。浏览器会刷新，不会触发onpopstate事件，每次点击都会刷新，会重新加载一次。

    • 点击浏览器的返回按钮不会触发onpopstate事件。

    注意：a标签的跳转引了hash的改变，在第一次操作会引起浏览器的刷新，不会触发hashchange事件和popstate事件，之后再操作会触发popstate事件，不会触发hashchange事件。

    # 应用

    # 场景1

    在实际应用中通常会遇到这样的需求，需要从一个目标页跳到另一个页面就行一些操作，操作完成再跳到目标页，如果这时候我们还使用pushState跳转到目标页，就会向history中再添加一条目标页的记录。

    如果我们在操作页跳转目标页的时候使用返回上一页history.go(-1)，这时只需要点击1次返回键就可以回到起始页，这样对于用户体验来说更合理一些。

    # 实现综合监听

    如果做到全面监听呢？这里要监听的是在浏览器不刷新的情况下发生的地址栏信息改变，所以先不考虑a标签的跳转。

    那么就只有下面几种情况：

    • pushState
    • replaceState
    • 浏览器的前进后退
    • history的 go back forward
        class ListenersURl {
        constructor() {
            this.oldUrl = location.href;
            this.newUrl = '';
            this.oldLocation = {...location};
            this.newLocation = {}
            this.listeners ={
                hash: [],
                history: []
            }
            console.log(this.oldUrl, location)
            this.init()
        }

        init() {
            const originPushState = window.history.pushState;
            const orignReplaceState = window.history.replaceState;


            window.history.pushState = (...args) => {
                originPushState.apply(window.history, args)
                this.changeUrl()
            }

            window.history.replaceState = (...args) => {
                orignReplaceState.apply(window.history, args)
                this.changeUrl()
            }

            window.addEventListener('popstate', () => {
                this.changeUrl()
            })
            window.addEventListener('hashchange', () => {
                this.changeUrl()
            })  
        }
        changeUrl() {
            const newUrl = location.href;
            if(newUrl !== this.oldUrl) {
                this.newUrl = newUrl;
                this.newLocation = { ...location };
                this.checkUrl()
            }
        }
        checkUrl() {
            const {oldLocation, newLocation} = this;
            const {hash, pathname} = oldLocation;
            const {hash: newHash, pathname: newPathname} = newLocation;
            if(hash !== newHash) {
                this.on('hash');
                return;
            }
            if(pathname !== newPathname) {
                this.on('history')
            }

        }
        addEventListener(name, fun) {
            const funs =  this.listeners[name] || []
            if(funs.indexOf(fun) > -1) return;
            funs.push(fun)
        }

        on(name) {
            this.oldLocation = {...location};
            this.oldUrl = location.href;
            const funs = this.listeners[name] || [];
            funs.forEach(fun => {
                fun(this.location)
            });
        }

    }
 
    const inst = new ListenersURl();
    inst.addEventListener('hash', function() {
        console.log('hashChange')
    })
    inst.addEventListener('history', function() {
        console.log('hsitoryChange')
    })
